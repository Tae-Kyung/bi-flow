# PRD: BI-Flow - 창업보육 통합 관리 플랫폼

## 1. 개요

### 1.1 제품 비전

BI-Flow는 창업보육센터(BI)의 입주·퇴거·정산 업무를 디지털화하는 **멀티 테넌트 SaaS 플랫폼**이다. 초기에는 3개 보육 기관(창업보육센터, G-테크벤처센터, 융합기술원)의 통합 관리를 목표로 하며, 장기적으로 전국 창업보육센터가 가입하여 사용하는 플랫폼으로 확장한다.

### 1.2 배경 및 문제 정의

| 문제 | 설명 |
|------|------|
| 수동 정산 | 엑셀 기반 임대료·관리비 산출로 오류 발생 및 시간 소모 |
| 분산 관리 | 3개 기관의 입주 현황·계약·수납 데이터가 각각 별도 관리 |
| 가시성 부족 | 기관별 입주율·미납률 등 핵심 지표를 실시간으로 파악할 수 없음 |
| 프로세스 비표준 | 입주 신청~퇴거까지 절차가 기관마다 상이 |

### 1.3 성공 지표

| 지표 | 목표 |
|------|------|
| 월 정산 업무 시간 | 기존 대비 70% 단축 |
| 고지서 발행 자동화율 | 100% |
| 입주 신청~승인 리드타임 | 기존 대비 50% 단축 |
| 미납 고지 누락률 | 0% |

## 2. 사용자 페르소나

### P1. 총괄 관리자

- **역할**: 3개 기관 전체를 관리하는 책임자 (교수님)
- **핵심 니즈**: 기관별 KPI 한눈에 파악, 기관 간 비교 분석
- **주요 시나리오**: 통합 대시보드에서 전체 입주율·수납률 확인, 기관 추가 등록

### P2. 기관 담당자

- **역할**: 개별 보육 기관의 실무 운영자
- **핵심 니즈**: 입주 신청 처리, 월별 정산, 계약 갱신 관리
- **주요 시나리오**: 입주 신청서 검토·승인, 월말 고지서 일괄 발행, 미납 기업 독촉

### P3. 입주기업 대표

- **역할**: 보육 기관에 입주한 스타트업/기업 대표
- **핵심 니즈**: 입주 신청, 고지서 확인, 납부 내역 조회
- **주요 시나리오**: 온라인 입주 신청서 제출, 대시보드에서 고지서 확인 및 납부

## 3. 시스템 아키텍처

### 3.1 멀티 테넌트 구조

```
┌──────────────────────────────────────────────────────┐
│                    Vercel (배포)                       │
│  ┌────────────────────────────────────────────────┐  │
│  │           Next.js App (Frontend + API)          │  │
│  │  ┌──────────┐  ┌──────────┐  ┌──────────┐     │  │
│  │  │ 창업보육  │  │G-테크벤처│  │ 융합기술원│     │  │
│  │  │ 센터     │  │ 센터     │  │          │     │  │
│  │  └────┬─────┘  └────┬─────┘  └────┬─────┘     │  │
│  └───────┼──────────────┼─────────────┼───────────┘  │
└──────────┼──────────────┼─────────────┼──────────────┘
           │              │             │
┌──────────┴──────────────┴─────────────┴──────────────┐
│                  Supabase (BaaS)                      │
│  ┌────────────┐ ┌──────────┐ ┌───────────────────┐   │
│  │ PostgreSQL  │ │   Auth   │ │     Storage       │   │
│  │ (RLS 적용)  │ │  (RBAC)  │ │ (서류 첨부파일)    │   │
│  └────────────┘ └──────────┘ └───────────────────┘   │
│  ┌────────────┐ ┌──────────────────────────────┐     │
│  │ Edge Func  │ │     Realtime (알림/구독)      │     │
│  └────────────┘ └──────────────────────────────┘     │
└──────────────────────────────────────────────────────┘
```

- **데이터 격리**: 기관별 `org_id`로 논리적 분리, Supabase RLS(Row-Level Security) 정책 적용
- **설정 기반 커스텀**: 기관마다 임대 단가, 관리비 배분 비율, 워크플로우를 독립 설정
- **실시간 동기화**: Supabase Realtime으로 대시보드 데이터 실시간 반영

### 3.2 역할 기반 권한 관리 (RBAC)

| 역할 | 권한 범위 |
|------|----------|
| Super Admin (총괄 관리자) | 모든 기관 데이터 조회/수정, 기관 등록, 시스템 설정 |
| Org Admin (기관 담당자) | 소속 기관 데이터 조회/수정, 입주 승인, 정산 관리 |
| Tenant (입주기업 대표) | 본인 기업 데이터 조회, 입주 신청, 고지서 확인 |

## 4. 핵심 기능 명세 (MVP)

### 4.1 입주 관리

#### 4.1.1 입주 신청

| 항목 | 내용 |
|------|------|
| 설명 | 입주기업이 온라인으로 입주를 신청하고, 담당자가 검토·승인하는 워크플로우 |
| 선행 조건 | 입주기업 대표가 회원가입 완료 |
| 기본 흐름 | 1. 기업 대표가 신청서 작성 (기업 정보, 희망 면적, 입주 기간) → 2. 서류 첨부 (사업자등록증, 사업계획서) → 3. 담당자 검토 → 4. 승인/반려 통보 |
| 산출물 | 입주 신청서 (디지털 아카이빙) |

#### 4.1.2 계약 관리

| 항목 | 내용 |
|------|------|
| 설명 | 입주 승인 후 임대차 계약서 생성 및 관리 |
| 기능 | 계약서 자동 생성 (템플릿 기반), 전자 서명 연동, 계약 기간 관리 |
| 알림 | 계약 만료 90일/60일/30일 전 자동 알림 (이메일 + 대시보드) |

#### 4.1.3 퇴거 관리

| 항목 | 내용 |
|------|------|
| 설명 | 퇴거 절차 체계화 및 졸업 기업 사후 관리 |
| 기능 | 시설 점검 체크리스트, 보증금 반환 처리, 미납금 정산 |
| 사후 관리 | 졸업 기업 DB 유지, 사후 추적 (매출·고용·투자 현황) |

### 4.2 스마트 정산 시스템

#### 4.2.1 임대료·관리비 산출

| 항목 | 내용 |
|------|------|
| 설명 | 기관별 설정에 따라 월 임대료와 관리비를 자동 산출 |
| 산출 로직 | 임대료 = 전용면적 × 면적당 단가, 관리비 = 공용면적 비율 배분 |
| 설정 항목 | 면적당 단가, 공용 관리비 총액, 배분 방식 (면적 비례/균등/커스텀) |

#### 4.2.2 고지서 발행

| 항목 | 내용 |
|------|------|
| 설명 | 매월 지정일에 입주기업에 고지서 자동 발행 |
| 발행 방식 | 입주기업 대시보드 게시 + 이메일 발송 + 알림톡(선택) |
| 고지서 내용 | 임대료, 관리비, 기타 부과금, 납부 기한, 가상계좌 정보 |

#### 4.2.3 수납 관리

| 항목 | 내용 |
|------|------|
| 설명 | 납부 상태 추적 및 미납 관리 |
| 상태 | 미발행 → 발행 → 납부완료 / 미납 → 연체 |
| 기능 | 입금 확인 처리 (수동), 미납 알림 자동 발송, 연체 현황 리포트 |

### 4.3 대시보드

#### 4.3.1 총괄 대시보드 (Super Admin)

- 전체 기관 요약: 총 입주기업 수, 평균 입주율, 총 미납 금액
- 기관별 비교 차트: 입주율, 수납률, 계약 만료 예정 건수
- 최근 활동 로그

#### 4.3.2 기관 대시보드 (Org Admin)

- 핵심 KPI: 입주율, 미납 현황, 계약 만료 예정 기업 수
- 월별 수납 추이 그래프
- 처리 필요 항목: 신규 입주 신청, 계약 갱신 대상, 미납 기업 목록

#### 4.3.3 입주기업 대시보드 (Tenant)

- 계약 정보 요약: 입주 기간, 전용면적, 월 임대료
- 고지서 목록 및 납부 내역
- 공지사항

| 구분 | 요구사항 |
|------|---------|

| 알림 | 이메일 필수, 알림톡(카카오) 선택 연동 |

## 7. 기술 스택

| 계층 | 기술 | 역할 |
|------|------|------|
| **Frontend** | Next.js 15 (App Router) + TypeScript | SSR/SSG, Server Actions, 풀스택 프레임워크 |
| **UI** | Tailwind CSS + shadcn/ui | 빠른 UI 개발, 일관된 디자인 시스템 |
| **Backend** | Supabase (BaaS) | DB, Auth, Storage, Realtime, Edge Functions |
| **DB** | Supabase PostgreSQL | RLS 기반 멀티 테넌트 데이터 격리 |
| **인증** | Supabase Auth | JWT 기반 인증, RBAC, 소셜 로그인 확장 가능 |
| **파일 저장** | Supabase Storage | 서류 첨부파일 (사업자등록증, 사업계획서 등) |
| **실시간** | Supabase Realtime | 대시보드 실시간 데이터 동기화 |
| **스케줄링** | Supabase Edge Functions + pg_cron | 고지서 자동 발행, 계약 만료 알림 |
| **배포** | Vercel | Next.js 최적 배포, Edge Network CDN |
| **알림** | Resend (이메일) | 고지서 발송, 계약 알림 등 이메일 발송 |
| **상태 관리** | Zustand / TanStack Query | 클라이언트 상태 및 서버 상태 관리 |

### 7.1 Supabase 활용 상세

| Supabase 기능 | 적용 영역 |
|---------------|----------|
| PostgreSQL + RLS | 기관별 데이터 격리 정책, 역할별 접근 제어 |
| Auth | 회원가입/로그인, JWT 발급, 역할(role) 관리 |
| Storage | 입주 신청 서류, 계약서 파일 저장 (버킷별 RLS 적용) |
| Realtime | 대시보드 KPI 실시간 업데이트, 알림 구독 |
| Edge Functions | 고지서 발행 배치, 외부 API 연동 (이메일, 알림톡) |
| Database Functions | 임대료·관리비 산출 로직, 복잡한 비즈니스 로직 |
| pg_cron | 월별 고지서 자동 발행, 계약 만료 알림 스케줄링 |

### 7.2 Vercel 활용 상세

| Vercel 기능 | 적용 영역 |
|-------------|----------|
| Edge Network | 정적 페이지 CDN 배포, 빠른 응답 속도 |
| Serverless Functions | Next.js API Routes (Supabase 연동 프록시) |
| Preview Deployments | PR별 미리보기 환경으로 QA 효율화 |
| Environment Variables | Supabase URL/키 등 환경 설정 관리 |
| Analytics | 웹 성능 모니터링 (Core Web Vitals) |

## 8. 마일스톤

> **전략 변경**: 정산 방식이 기관별로 상이하여 단일 시스템으로 조기 표준화가 어렵다고 판단.
> 입주·퇴거 프로세스와 기업 현황 조회를 먼저 완성한 뒤, 정산 시스템을 후속 추가한다.

### Phase 1 - Foundation ✅

- 사용자 인증 및 RBAC
- 기관(Organization) 등록 및 관리
- 공간/호실 관리
- 입주기업 등록 및 기본 정보 관리

### Phase 2 - 입주·퇴거 프로세스 + 기업 현황

- 온라인 입주 신청 워크플로우 (신청→서류→검토→승인/반려)
- 서류 업로드 및 디지털 아카이빙
- 계약서 생성 및 관리, 계약 만료 알림
- 퇴거 관리 (퇴거 신청→시설 점검→보증금 반환→졸업 처리)
- 기업 현황 대시보드 (입주율, 계약 현황, 입주기업 상태 요약)

### Phase 3 - 대시보드 고도화

- 총괄/기관/입주기업별 대시보드
- KPI 시각화 (입주율 추이, 기관별 비교)
- 계약 만료 예정·졸업 기업 추적 리포트

### Phase 4 - 정산 시스템 (기관별 규칙 수용)

- 기관별 정산 규칙 설정 (단가, 배분 방식, 부과 항목 커스텀)
- 임대료·관리비 자동 산출
- 고지서 발행 및 수납 관리
- 미납 알림

### Phase 5 - 확장 (Post-MVP)

- 전자 서명 연동
- 알림톡(카카오) 연동
- 회계·ERP 연동
- 전국 확산을 위한 셀프 온보딩
